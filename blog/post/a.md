### 담당역할 및 개발사항
1. **근로자 CRUD 화면**
- 개요
- 신규 근로자의 신상 정보를 등록 수정 삭제하는 기능을 만들고, 부서에 따라
  사원코드의 앞자리가 다르게 생성되어야 하며, 중복되어서는 아니된다.
- 개발방법
- 입력창을 먼저 보여주고 선택 시 모달창을 띄워 정보를 입력하게 만든다.
- 부서 선택에 따른 조건문을 만들어 사원 코드 생성 기능을 만든다.
- 중복 방지를 위해 부서 전체 데이터를 조회하고 생성되게 만든다.
- 개발간 문제점
- 넥사크로는 받아오는 값을 미리 데이터베이스에 등록해야 데이터 수신이 가능해진다.
  그래서 넥사크로 데이터베이스에 수많은 칼럼을 등록해야 했기 때문에 많은 시간이 소요되었다.
- 고유한 트랜잭션을 통해서 데이터를 주고 받는다. 배운 적 없는 방법이었기 때문에 이해가 어려웠다.
- 자료가 충분하지 않아 넥사크로 커뮤니티나 회사의 소개영상으로 공부해야 했다.
- 개발간 문제 해결 방법
- 무슨 기능이 어디에 있어야 하는 지 알아야 했기 때문에 학원의 개발 표준안에 따라 천천히 진행했고, 데이터
  모델과 유스케이스를 직관적으로 보기 위해 타사의 프로그램을 참고해서 제작하였다.
- 개발간 느낀점
- 첫 프로젝트다 보니 기능을 완성하기는 커녕 데이터를 받아오는 것만으로도 감격스럽다.
  회사의 영상을 보고 이해를 한 뒤 기능을 만들어야 했기 때문에 개인의 역량에 따라 맡은 업무의
  진행에 차질이 생기기도 했다. 하지만 사람이 여럿 있으니 각자의 정보를 종합해서 문제를
  해결할 수 있었다.

## React

두 번째는 리액트를 활용해 개발표준안을 확인하고 데이터 그리드와 사이드 바를 활용해서 필요한 기능을
곧장 찾을 수 있는 화면을 개발했습니다.

### 프로젝트 개요
- 기간: 2024.02 ~ 2024.05
- 인원: 4명(FE&BE:3명, 이탈:1명)
- 기술스택
- FE: Next.js, React, TypeScript, Material-UI, Redux
- BE: SpringBoot
- DB: Oracle
- ETC: Jira, ESLint, UML(UseCase Diagram)
### 기술스택 선정 사유
#### <div style={{ color: 'red' }}>Next.js</div>
- 클라이언트 요청으로 Berry 탬플릿을 Next.js를 사용해 만들게 되었다.
#### <div style={{ color: 'red' }}>TypeScript</div>
- 클라이언트 요청으로 전체적인 안정성을 높이기 위해서 채택되었다.
#### <div style={{ color: 'red' }}>Material-UI</div>
- Berry 탬플릿이 MUI를 사용해서 만들어졌기 때문에 이를 활용하기 위해서 배워야 했다.
#### <div style={{ color: 'red' }}>Oracle</div>

- 팀원과의 소통과 업무의 진행도를 파악하기 위해 선택되었다.

1. **기획과정**
- 개발표준안
2. **설계과정**
- UML - UseCase Diagram

### 프로젝트 기능
1. **회계전기**
- 거래관리: 차변과 대변을 기입해서 전기를 작성한다.
- 회계장부관리: 전표를 관리할 수 있는 기능을 만들어야 한다.
2. **결산마감**
- 시산표 관리: 전기의 데이터를 계산해서 시산표에 기입한다.
- 재무제표관리: 수익과 비용을 계산하여 포괄손익계산서를 작성한다.
3. **예산관리**
- 예산 편성: 현금 흐름표를 그래프로 나타낼 수 있어야 한다.
### 담당역할 및 개발사항
1. **기초정보 관리**
- 개요
- 신규 거래처 등록 시 거래처 코드가 생성되어야 하고 거래 내용을 입력할 수 있어야 한다.
- 개발방법
- 거래처 코드에 PK값을 부여하여 코드가 중첩되지 않게 만들고 CRUD를 작성한다.
- 개발간 문제점
-  CORS 에러를 확인했다.
- 개발간 문제 해결 방법
-  스프링 addCorsMappings의 allowOrigins에 포트번호를 추가했다.

2. **전표 등록**
- 개요
- 거래 데이터를 받아와 전표를 작성할 수 있어야 한다.
- 개발방법
- 거래 데이터를 검색하는 기능을 만들고 셀을 선택하면 모달창이 나오며 해당 거래처의 세부데이터가 나오게 만들어야 한다.
- 개발간 문제점
-  화면이 시작될 때 데이터를 받아오고 모달창 선택 시 따로 데이터를 불러와야 하며 데이터 선택시 자동으로 거래가 기입되게 만들어야 했다.
-  모달창을 열 때마다 데이터를 받아오니 속도가 느려졌다.
- 개발간 문제 해결 방법
-  async/await를 사용해서 데이터를 미리 받아오는 작업을 진행하니 모달창의 출력 속도가 빨라졌다.
- 개발간 느낀점
- 완전 자동화 기능을 만드는 게 아니라서 대부분 CRUD를 작성하는 단순작업의 반복이었다. 마이바티스가 사용하기는 쉬워도
  개발 속도에 있어서 한계점이 생긴다는 것을 알게 되었다. 개발 초기 단계에 어떤 방식으로 개발을 진행할지
  더 깊게 생각했어야 했다.
- 중간에 한 명이 학원을 그만두게 되었다. 아무래도 개인의 역량 차이도 있으니 그럴 수 있다고 생각했다. 아는게 많았으면
  도와줄 수 있었겠지만 부족한 입장이라 손을 쓸 수 없어 아쉬웠다.

### 프로젝트 기능
1. **사원 등록**

### 담당역할 및 개발사항
1. **로그인**
- 개요
- 사원의 로그인 기능을 구현해야 한다.
- 개발방법
- 로그인 아이디를 사원 번호로 설정하고 Base64 코드 변환을 이용해서 비밀번호를 데이터베이스에 저장한다.
- 로그인 시 유저 프로필이 변경되어야 한다.
- 개발간 문제점
- 스프링 시큐리티를 적용해야 했으며 백앤드에서 Base64 인코딩을 해야했다.
- 개발간 문제 해결 방법
- 2주일 조금 넘는 시간을 사용해서 시큐리티 기초를 배우고 로그인 로직을 구현했다.
2. **근태**
- 개요
- 출근,기타,퇴근 기능을 만들고 근로 시간을 입력해야 한다.
- 개발방법
- 근로자의 출근 시간과 퇴근 시간 사이의 값을 계산해서 근로시간을 입령할 수 있어야 하고 외출,교육 같은
  상황 발생시 승인 여부에 따라 근로시간이 변경되어야 한다.
- 개발간 문제점
- 해당 기능을 구현하던 인원이 학원을 그만둬서 최대한 빠르게 구현해야했다.
- 개발간 문제 해결 방법
- 1시간 단위로 근로시간을 책정하게 만들었고,데이터 입력 후 특수 상황 발생시 승인을 받으면 트리거로 처리했다.
3. **급여**
- 개요
- 근태 기록과 사원 개인 정보에 따라 급여를 계산하게 만들어야 했다.
- 개발방법
- 근태에서 담당했던 기능과 연결되는 기능이었기 때문에 사원 데이터만 따로 받아와서 처리되게 만들었다.
- 개발간 문제점
- 기능을 만드는 것에는 문제가 없었는데, 한번에 많은 양의 데이터를 불러오게 되어서 그리드의 가독성이 떨어지게 되었다.
- 개발간 문제 해결 방법
- 그리드에 검색기능을 도입해서 부서와 사원으로 구분할 수 있게 만들어 가독성을 높였다.
- 개발간 느낀점
- 인원이 부족한 상황에서 만들다 보니 개인 성향이 짙게 나타나게 되었다.
- 인원 감축으로 인한 시간 문제로 개성적인 코드가 생성되다 보니 개발표준안을 무시하고 매핑한 것들이 심심찮게 보였다.
